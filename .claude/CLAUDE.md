# Claude Code — Инструкции для проекта

Ты редактируешь премиальный лендинг для **AI FOR CREATORS 4.0**.

---

## Перед изменениями

1. **Прочитай**:
   - `website-guidelines/01_style-guide.md` (палитра, типографика, компоненты)
   - `website-guidelines/02_component-map.md` (где что искать)
   - `website-guidelines/03_copy-guide.md` (правила текста)

2. **Найди секцию**:
   - Открой `site/index.html`
   - Найди секцию по ID (`#hero`, `#author`, `#cases`, и т.д.)
   - Используй Cmd+F для поиска по классу или тексту

3. **Проверь стили**:
   - Токены в `site/styles/custom.css` → `:root`
   - Компоненты там же (`.btn-primary`, `.input-field`)
   - Tailwind классы в HTML

---

## Правила

### Дизайн
- ✅ Только цвета из палитры (`01_style-guide.md`)
- ✅ Hairline borders (1px solid `--line`)
- ✅ Grayscale изображения (hover → цвет)
- ❌ Нет SaaS-градиентов
- ❌ Нет новых акцентных цветов
- ❌ Нет толстых рамок или теней

### Контент
- ✅ Весь текст на русском
- ✅ CTA везде одинаковый: "Анкета предзаписи"
- ✅ Editorial/zine тональность (коротко, остро, без клише)
- ❌ Нет инфобизнес-фраз ("Уникальная возможность", "Успей")
- ❌ Нет пафоса и "воды"

### Код
- ✅ Стили в `custom.css`, не инлайн
- ✅ ID для секций, классы для стилизации
- ✅ Accessibility (alt, aria-required, labels)
- ❌ Не трогай `source/generated-page.html` (это архив)
- ❌ Не добавляй внешние зависимости без обсуждения

---

## Где править

| Задача | Файл | Что искать |
|--------|------|-----------|
| Текст hero | `site/index.html` | Секция `#hero`, строки ~100-120 |
| Цвета | `site/styles/custom.css` | `:root` CSS variables |
| Кнопки | `site/styles/custom.css` | `.btn-primary`, `.btn-ghost` |
| Форма | `site/index.html` | Секция `#presale`, строки ~420-450 |
| Обработка формы | `site/scripts/main.js` | `initPresaleForm()` |
| FAQ | `site/index.html` | Секция `#faq`, элементы `<details>` |
| Фото автора | `site/index.html` | Секция `#author`, `src="assets/images/..."` |

## Workflow

1. **Читаешь задачу** от пользователя
2. **Проверяешь гайдлайны** (`website-guidelines/`)
3. **Находишь нужный файл** (чаще всего `site/index.html` или `custom.css`)
4. **Делаешь минимальные изменения** (не ломай работающее)
5. **Проверяешь**:
   - Палитра не нарушена?
   - Текст на русском?
   - CTA не изменился?
   - Accessibility не пострадал?

---

## Output

- **Показываешь только изменённые файлы** (не весь проект)
- **Объясняешь**, что и зачем изменил
- **Копи остаётся RU**, editorial/zine тональность
- **Стиль остаётся консистентным** с существующим

---

## Не делай

- ❌ Не добавляй цвета вне палитры
- ❌ Не переводи на английский
- ❌ Не создавай новые секции без запроса
- ❌ Не используй инфобизнес-клише
